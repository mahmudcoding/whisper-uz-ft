1. PROJECT OVERVIEW

This repository builds a dedicated open-weight Uzbek ASR model based on
openai/whisper-large-v3. The objective is to minimize Uzbek WER/CER. Multilingual
capability is not a goal, and catastrophic forgetting of non-Uzbek languages is
acceptable when it improves Uzbek.

The project contains data acquisition/preparation, Uzbek text normalization,
deduplication, quality scoring, LR/freeze-boundary search, training, evaluation,
inference benchmarking, and capacity planning.

6. NORMALIZATION

The production normalizer is src/text_normalization/uz_normalizer.py.
Canonical target is Uzbek Latin. It normalizes Unicode, lowercasing, apostrophe variants,
Cyrillic-to-Latin mappings, punctuation, and whitespace. Important mappings include
ў -> o', қ -> q, ғ -> g', ҳ -> h, ч -> ch, ш -> sh, ю -> yu, я -> ya.

Known limitations include incomplete number verbalization and unresolved policy for
some Russian code-switch spellings.

7. TRAINING PIPELINE

Core files:
- src/model.py loads Whisper large-v3, enforces Uzbek transcribe, sets forced decoder
  prompt IDs, and applies tuning modes.
- src/train.py loads CSVs, extracts audio features, builds custom AdamW parameter
  groups, configures Seq2SeqTrainer, computes WER/CER/hallucination/language-confusion,
  writes JSON metrics, writes status reports, validates checkpoints, and saves best
